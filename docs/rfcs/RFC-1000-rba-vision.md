# RFC-1000: RBA Vision

Status: Draft
Version: 0.1
Author: RBASIC Project
Created: 2026-06-17
Last Updated: 2026-06-17

---

## 1. Resumen

Este documento describe la visión a largo plazo para **RBA (RBASIC for Applications)**, una plataforma de automatización basada en RBASIC diseñada para integrarse con suites ofimáticas modernas (LibreOffice, ONLYOFFICE, etc.) y ofrecer una alternativa segura y portable a VBA.

---

## 2. Motivación

VBA permite automatizar documentos y hojas de cálculo, pero sufre de dependencias propietarias, sintaxis obsoleta, falta de tipado y seguridad limitada. RBA busca resolver estos problemas usando RBASIC como núcleo.

---

## 3. Relación entre RBASIC y RBA

```
RBASIC Core
      ↓
RBA Runtime
      ↓
Office Integration Layer
      ↓
LibreOffice / ONLYOFFICE
```

RBASIC es el lenguaje base; RBA es una especialización para automatización de aplicaciones.

---

## 4. Objetivos

**Objetivo principal**: crear una plataforma moderna de automatización para documentos y hojas de cálculo.

**Objetivos secundarios**
- Reemplazar progresivamente casos de uso tradicionales de VBA.
- Mantener una sintaxis familiar para usuarios de VBA.
- Proveer APIs seguras y tipadas.
- Permitir portabilidad entre suites ofimáticas.
- Integrar capacidades modernas de desarrollo.

---

## 5. Principios

1. **Compatibilidad conceptual** – los usuarios de VBA reconocerán fácilmente los conceptos.
2. **Seguridad por defecto** – sandboxing, permisos explícitos, APIs seguras.
3. **Portabilidad** – el mismo código debe ejecutarse en distintas plataformas.
4. **Tipado moderno** – RBA heredará el sistema de tipos de RBASIC.

---

## 6. Casos de uso

### Hojas de cálculo
```basic
SUB Main()
    LET sheet = Workbook.ActiveSheet
    sheet.Range("A1").Value = "Hello RBA"
END SUB
```

### Procesamiento de documentos
```basic
SUB Main()
    LET doc = Application.ActiveDocument
    doc.AppendParagraph("Generated by RBA")
END SUB
```

### Automatización empresarial
```basic
SUB Main()
    LET total = Sheet.Range("B10").AsNumber()
    IF total > 1000 THEN
        PRINT "Threshold exceeded"
    END IF
END SUB
```

---

## 7. Modelo de APIs

Objetos estándar previstos:
```
Application, Workbook, Worksheet, Range, Cell, Document, Paragraph, Table, Chart, Shape, Selection
```
Cada API será detallada en RFCs específicas.

---

## 8. Formatos de archivo

- **RBASIC scripts**: `.rbas`
- **RBA scripts**: `.rba`

---

## 9. Arquitectura propuesta
```
RBA Script → RBASIC Compiler → RBA Runtime → Office Adapter → LibreOffice / ONLYOFFICE
```

---

## 10. Seguridad

Modelo de permisos declarativos, por ejemplo:
```basic
REQUIRES SpreadsheetAccess
```
Posibles permisos: `SpreadsheetAccess`, `DocumentAccess`, `FileSystemAccess`, `NetworkAccess`, `ClipboardAccess`.

---

## 11. Roadmap
1. **Fase 1** – RBASIC Core estable.
2. **Fase 2** – Runtime básico.
3. **Fase 3** – Sistema de módulos.
4. **Fase 4** – API de automatización.
5. **Fase 5** – Adaptador LibreOffice.
6. **Fase 6** – Adaptador ONLYOFFICE.
7. **Fase 7** – Ecosistema RBA completo.

---

## 12. Fuera del alcance actual
RBA no forma parte de RBASIC v0.1 y no será implementado durante los hitos de lexer, parser, semantic analyzer, backend (Rust/rustc) o runtime básico.

---

## 13. Declaración de visión
RBA aspira a ser una plataforma moderna, segura y portable de automatización de documentos y hojas de cálculo basada en RBASIC, proporcionando una alternativa abierta a VBA para la próxima generación de herramientas ofimáticas.

Category: Strategic Vision

Note: RFC-1000 does not affect RBASIC v0.1 implementation. This document describes a long‑term ecosystem vision and must not be used as a source of requirements for current milestones.
