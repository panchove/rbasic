# v0.3 RFC Roadmap

## Planned RFCs (in order)

1. **RFC-0019 INPUT Statement**
   - Purpose: Add interactive user input.
   - Reason: Basic I/O capability needed for many programs.
   - Dependencies: Existing lexical/parser, primitive types.
   - Out of scope: Complex format strings.
   - Risk: Low.

2. **RFC-0020 SELECT CASE**
   - Purpose: Structured multi‑branch control flow.
   - Reason: Improves readability for many‑way decisions.
   - Dependencies: Parser extensions for `SELECT`, `CASE`, `END SELECT`.
   - Out of scope: Pattern‑matching (`CASE IS`, `CASE ELSE`).
   - Risk: Medium.

3. **RFC-0021 SUB Procedures**
   - Purpose: Introduce sub‑routines without return values.
   - Reason: Aligns with classic BASIC and supports code reuse.
   - Dependencies: Symbol table handling for procedures, call‑stack semantics.
   - Out of scope: Recursive sub‑procedure calls.
   - Risk: Medium.

4. **RFC-0022 Module System**
   - Purpose: Provide namespacing and separate compilation units.
   - Reason: Prepares the compiler for larger projects and library sharing.
   - Dependencies: Module import syntax, separate symbol tables per module.
   - Out of scope: Dynamic loading or runtime linking.
   - Risk: High.

5. **RFC-0023 Function References**
   - Purpose: Allow passing functions as first‑class values.
   - Reason: Needed for higher‑order constructs used by later features.
   - Dependencies: Function identifier resolution, type `FuncRef`.
   - Out of scope: Full closure support (captured environments).
   - Risk: High.

---

## Deferred Beyond v0.3

### RFC-0024 Semantic Information Model
- **Purpose:** Formalise a future `SemanticInfo` structure that aggregates arrays, globals, functions, inferred types, etc.
- **Status:** Deferred.
- **Reason:** Current architecture does not require it; observations ARCH‑002 and ARCH‑003 document the long‑term direction.

### RFC-0025 Intermediate Representation Architecture
- **Purpose:** Introduce a backend‑agnostic IR to decouple semantic analysis from code generation.
- **Status:** Deferred.
- **Reason:** No IR is needed for v0.2 or v0.3; see observations ARCH‑003.

These RFCs become relevant only after the v0.3 features (INPUT, SELECT CASE, SUB, Module System, Function References) are implemented and stabilised.
